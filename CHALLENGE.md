# Clutch Full-Stack Developer Challenge V2

> Local copy of the [original brief](https://clutchapp.notion.site/Clutch-Full-Stack-Developer-Challenge-V2-31ff733f6e6980cd9674f72d0f36b19e) in case the link expires.

## Task

Build a simple mobile application using React Native and a back-end service of your choice. The app you build will need to have the following features:

- A basic user registration and login/logout flow (i.e. username/password). Authentication state should persist.
- A dashboard for viewing videos on a feed, similar to Instagram.
- Users should be able to "like" videos on the feed, with a counter displaying number of unique likes per video.

Treat this application as the foundation of a production-ready system, with careful consideration given to scalability, maintainability, and future growth.

## Specifications

- Videos for the feed must be fetched from this API: `https://jslnnchztrhhrzytyzql.supabase.co/functions/v1/clutch-demos`
- Please host the code in a public GitHub repository, and make incremental commits to it that reflect your development process as though you were working on it collaboratively
- You are encouraged to use Supabase for your back-end (It's our back-end service provider)
- You are encouraged to install and utilize 3rd party libraries
- You may use AI coding assistants, however you must demonstrate a clear understanding of any code submitted
- You may use mock data where necessary

## Submission & Timeline

- You have 4 days from receiving this link to complete the challenge. Note: this timeline is intended as a buffer — we don't expect you to spend an excessive amount of time on it. Please complete as much as needed to confidently showcase your abilities.
- Upon completion, please send us a link to the Github repository, alongside any environment variables required to run the project locally
- You will be required to demonstrate your app and walk us through your logic and thought processes.

## Bonus Challenges

If you would like to add some extra features to the app, here are some suggestions:

**EASY:** Allow users to upload and edit profile pictures

**MEDIUM:** Each entry of the video API response provided includes 3 video URLs; allow users to toggle between these 3 videos

**HARD:** Add additional social media features to the feed: comments, a "Share" button, a list of users who liked the video, etc.

Of course, feel free to add anything else you might see fit.
